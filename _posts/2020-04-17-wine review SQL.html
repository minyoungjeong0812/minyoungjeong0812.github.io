From this we can see that there are two tables in this dataset: (1) winemag_first150k and (2)winemag_130k_v2. I will use the first table only for this data analysis.

To see how many instances are 'empty' in each column in the original data table, I made the following SQL query. Basically I am assigning '1' to null values and '0' to non-null values and summing up the values across the rows. If the summed up value is 0 for a certain column, it means all rows in that specific column does NOT have any null values.

In [13]:
%sql SELECT sum(case when country is null then 1 else 0 end) as country,sum(case when price is null then 1 else 0 end) as price,sum(case when province is null then 1 else 0 end) as province, sum(case when region_1 is null then 1 else 0 end) as region_1, sum(case when region_2 is null then 1 else 0 end) as region_2, sum(case when variety is null then 1 else 0 end) as variety, sum(case when winery is null then 1 else 0 end) as winery from winemag_first150k

We can clearly see that the population follows some bell-shaped normal distribution as expected. Overall we don't expect there will be a ton of 'perfect wines' and also a ton of 'low quality wines'.

Also, the average price becomes more expensive as the points go up. This makes sense as you do would not pay more for lower quality ones :)
